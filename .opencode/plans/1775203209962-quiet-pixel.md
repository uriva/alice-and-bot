# Preact to Lit Migration Plan

## Goal
Migrate the chat UI from Preact to Lit while preserving behavior. Use the completed Playwright E2E suite as the regression safety net and keep backend, protocol, auth, and schema behavior unchanged.

## Scope
- `clients/react/src/abstractChatBox.tsx`
- `clients/react/src/main.tsx`
- `clients/react/src/hooks.ts`
- `widget/src/widget.tsx`
- `widget/src/entry.tsx`
- `landing/src/chat.tsx`

## Non-goals
- No protocol/backend rewrite
- No schema or permissions changes
- No InstantDB data model changes
- No auth model changes

## Key Decision
Do not port `@instantdb/react` hooks to Lit. Replace the hook layer with a framework-agnostic layer built on `@instantdb/core` / Instant vanilla APIs:
- `init({ appId, schema })`
- `db.subscribeQuery(...)`
- `db.subscribeInfiniteQuery(...)`
- `db.transact(...)`
- `db.room(type, id)`

This is the main architectural move. The UI port is secondary.

## Current Architecture

### 1. `AbstractChatBox`
- Heavy presentational component with local interaction state
- Uses `useState`, `useEffect`, `useMemo`, `useRef`
- Owns input behavior, markdown/code rendering, edits, audio/video players, attachment rendering, copy UI, typing/progress/spinner visuals
- Migration difficulty: medium-high

### 2. `Chat`
- Container/orchestrator around `AbstractChatBox`
- Owns conversation key lookup, decrypted messages, typing presence, identity details, ephemeral streams, send/edit/upload handlers, voice call integration
- Depends heavily on hook layer in `hooks.ts`
- Migration difficulty: high

### 3. `hooks.ts`
- Current React/Preact integration layer
- Owns all Instant subscriptions and a lot of lifecycle orchestration
- Main functions: `useConversations`, `useConversationKey`, `useDecryptedMessages`, `useTypingPresence`, `useEphemeralStreams`, credentials/localStorage helpers, mobile/dark mode helpers
- Migration difficulty: very high

### 4. `Widget`
- Wraps Chat in a shadow-DOM-based widget shell
- Uses signals, portals, effects, refs, localStorage, mobile viewport logic, name dialog
- Migration difficulty: high

### 5. `Messenger`
- Full page shell with login, credential handling, conversation list, route sync, profile/settings, invite flow
- Largest state surface
- Migration difficulty: very high

## Phase 0
Already complete:
- E2E suite implemented and passing before migration work
- Tests are framework-agnostic and should remain unchanged during migration

## Phase 1: Build a Lit-friendly data layer

### Goal
Replace the current Preact hook layer with framework-agnostic subscription primitives.

### New files
- `lit/core/instant-client.ts`
- `lit/core/subscriptions.ts`
- `lit/core/credentials.ts`
- `lit/core/room.ts`
- `lit/core/store.ts`

### Responsibilities

#### `instant-client.ts`
- Export singleton/core Instant instance via `@instantdb/core`
- Own `init({ appId, schema })`
- Centralize room creation

#### `subscriptions.ts`
Provide pure subscription APIs that return unsubscribe handles, for example:
- `subscribeConversations(publicSignKey, onChange)`
- `subscribeConversationKey(conversationId, publicSignKey, onChange)`
- `subscribeDecryptedMessages(conversationId, conversationKey, onChange)`
- `subscribeIdentityDetails(publicKeys, onChange)`
- `subscribeTypingPresence(conversationId, selfPublicSignKey, onChange)`
- `subscribeEphemeralStreams(conversationId, onChange)`

Rules:
- use `db.subscribeQuery` / `db.subscribeInfiniteQuery`
- do transformation and decryption before notifying UI
- keep unsubscribe explicit

#### `credentials.ts`
- load/save credentials from localStorage
- create identity
- import/export credentials

#### `room.ts`
- wrap `db.room(...)`
- presence + topics wiring
- typing + stream publishing/subscribing

#### `store.ts`
- tiny framework-agnostic state primitive with `get`, `set`, `subscribe`
- keep it minimal; avoid framework coupling

### Acceptance criteria
- No Lit components yet
- The new layer can provide the same data as `hooks.ts`
- No dependency on `@instantdb/react` in the new path

## Phase 2: Port `AbstractChatBox` to Lit

### Goal
Port the big UI component first while keeping its public API close to the current one.

### New files
- `lit/components/abstract-chat-box.ts`
- possibly extracted pieces:
  - `lit/components/chat-avatar.ts`
  - `lit/components/code-block.ts`
  - `lit/components/audio-player.ts`
  - `lit/components/location-card.ts`
  - `lit/components/attachment-renderer.ts`
  - `lit/components/message-actions.ts`

### Lit mapping
- props -> `@property`
- internal state -> `@state`
- refs -> `@query` / `@queryAll`
- effects -> `firstUpdated`, `updated`, `disconnectedCallback`
- derived values -> getters

### Decision
Start with a mostly API-compatible Lit version instead of redesigning the interface.

### Acceptance criteria
- `e2e/abstract-chat-box.spec.ts` passes unchanged
- light/dark mode parity preserved
- all current `data-testid` selectors preserved

## Phase 3: Port `Chat` to Lit

### Goal
Replace the current orchestrator with Lit plus the new core subscription layer.

### New files
- `lit/components/chat.ts`
- `lit/controllers/chat-controller.ts`

### Controller responsibilities
- subscribe/unsubscribe conversation data
- subscribe/unsubscribe conversation key
- subscribe/unsubscribe decrypted/infinite messages
- subscribe/unsubscribe identity details, typing presence, streams
- expose send/edit/upload/loadMore actions to host component

### Lit state to own
- `conversation`
- `conversationKey`
- `messages`
- `typingNames`
- `identityDetails`
- `activeSpinners`
- `activeProgress`
- `activeStreams`
- `hasAccess`
- loading/error state

### Acceptance criteria
- `e2e/chat.spec.ts` passes unchanged
- no migrated Chat path depends on `@instantdb/react`

## Phase 4: Port `Widget`

### Goal
Replace the Preact widget shell with a Lit custom element.

### New files
- `lit/components/widget.ts`
- `lit/components/name-dialog.ts`

### Simplification opportunity
Current widget manually creates a shadow root and portals into it. Lit can own the custom element and shadow root directly, so the migration should remove complexity rather than translate it.

### Keep behavior
- start button / custom button text
- open/close/re-open
- name dialog
- escape behavior
- mobile fullscreen behavior
- localStorage credential reuse

### Acceptance criteria
- `e2e/widget.spec.ts` passes unchanged
- shadow DOM behavior preserved

## Phase 5: Port `Messenger`

### Goal
Port the full page shell last, after `Chat` is stable.

### Recommended split before or during port
- `lit/components/messenger-shell.ts`
- `lit/components/messenger-login.ts`
- `lit/components/messenger-chat-list.ts`
- `lit/components/messenger-profile.ts`
- `lit/components/messenger-danger-zone.ts`

### Keep behavior
- credential bootstrap from localStorage
- chat-with invite flow
- selected conversation URL sync
- identity/profile editing
- conversation list + open conversation
- dark mode toggle
- transfer import flow

### Recommendation
Keep the outer route shell as-is temporarily if needed, and mount Lit inside it first. Remove remaining Preact shell later.

### Acceptance criteria
- `e2e/messenger.spec.ts` passes unchanged

## Phase 6: Cleanup
- delete obsolete migrated Preact paths
- remove `@instantdb/react` usage from migrated code paths
- keep only framework-agnostic helpers that still make sense

## Exact execution order
1. Build `@instantdb/core` subscription layer
2. Port `AbstractChatBox`
3. Port `Chat`
4. Port `Widget`
5. Port `Messenger`
6. Remove migrated Preact-specific glue

## Risks and mitigations

### Risk: subscription lifecycle bugs
- Mitigation: central controller/store layer with explicit unsubscribe cleanup

### Risk: infinite query behavior mismatch
- Mitigation: implement and verify `subscribeInfiniteQuery` semantics before Chat migration

### Risk: shadow DOM styling differences in Widget
- Mitigation: preserve widget tests and validate mobile/fullscreen behavior at each step

### Risk: modal/overlay regressions
- Mitigation: preserve existing E2E behavior for dialog open/close/escape/reopen

### Risk: too much change at once
- Mitigation: keep E2E green after each phase; do not batch multiple components together

## Critical files expected to change during implementation
- `clients/react/src/abstractChatBox.tsx` or replacement wrapper path
- `clients/react/src/main.tsx`
- `clients/react/src/hooks.ts`
- `widget/src/widget.tsx`
- `widget/src/entry.tsx`
- `landing/src/chat.tsx`
- new `lit/` directory

## Verification

### Per phase
- run only the relevant suite first:
  - `deno run -A --node-modules-dir=auto npm:playwright test abstract-chat-box.spec.ts`
  - `deno run -A --node-modules-dir=auto npm:playwright test chat.spec.ts`
  - `deno run -A --node-modules-dir=auto npm:playwright test widget.spec.ts`
  - `deno run -A --node-modules-dir=auto npm:playwright test messenger.spec.ts`

### Final verification
- `deno task test:e2e`
- verify all suites pass unchanged on the Lit implementation

## Recommended first implementation slice
1. add `lit/core/instant-client.ts`
2. add `lit/core/subscriptions.ts` with `subscribeConversationKey`
3. add `lit/core/subscriptions.ts` with `subscribeDecryptedMessages`
4. port `AbstractChatBox` to Lit with API compatibility
5. create temporary Lit `Chat` shell using static props first
6. wire real subscriptions into Lit `Chat`

## Done criteria
- all four E2E suites pass on Lit
- migrated path no longer depends on `@instantdb/react`
